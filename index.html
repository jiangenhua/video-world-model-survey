第二章：现状分析

2.1 数据规模与时长分布

当前开源数据集在规模上呈现显著的两极分化。VidChapters-7M (~307,000h) 和 Action100M (~128,000h) 在总时长上遥遥领先，但并非为世界模型训练设计。适合世界模型训练的"中等时长（1-10min）、大规模"数据集仍然稀缺。

2.2 领域与场景分布

当前数据集在 domain 分布上存在明显偏向：游戏类占据主导（26.1%），日常真实场景类偏少且碎片化（17.4%），具身智能类规模偏小（合计不足 130 小时）。

2.3 数据来源与质量

YouTube 抓取（43.5%）是最普遍的数据来源，优势在于成本低、规模大、场景多样，但无法获得 GT 标注。引擎渲染（26.1%）能提供精确 GT 但存在 domain gap。专业设备采集（21.7%）质量最高但规模受限。

2.4 标注体系现状

第三章：痛点深度剖析

3.1 Action 标注严重缺失

核心痛点：73.9% 的数据集完全无 Action 标注

Action 标注对于 action-conditioned generation 至关重要，但 23 个数据集中仅 6 个提供了某种形式的 action 标注，其中 GT 标注仅有 4 个（含 WildWorld）。这直接制约了世界模型的交互性建模能力。

3.2 Camera Trajectory 标注不统一

标准不统一，精度差异大

Camera Trajectory 覆盖率虽达 56.5%，但不同来源（UE5 GT / AirSim / COLMAP / ViPE / MegaSaM）在坐标系、尺度归一化、内参格式上缺乏统一标准，严重阻碍跨数据集联合训练。

第四章：社区缺口分析

缺口一：日常真实场景 + 完备标注

目前没有一个日常真实场景数据集同时具备 caption + camera trajectory + action 标注。VIPE/SpatialVID 无 action，MiraData/DL3DV 标注不完整。

缺口二：Caption + Camera + Action "三位一体"

仅 GF-Minecraft（70h, Minecraft）和 WildWorld（1,000h, 游戏）具备。真实世界场景中此组合完全空白。

缺口三：帧级细粒度 Action 分类体系

除 WildWorld（450+ types, 逐帧 triplet）和 EPIC-KITCHENS-100（动词+名词）外，其他 action 标注均为粗粒度文本描述。

缺口四：中等时长 + 大规模 + 多维标注

在 100-10,000 小时区间内，同时具备三大核心 feature 的"标注完备型"数据集几乎空白。

第六章：总结与建议

核心发现

  1. Action 标注严重缺失是当前最大瓶颈——73.9% 的数据集完全无 action 标注，直接制约 action-conditioned generation 研究。
  2. 多维标注碎片化普遍存在——同时具备 caption + camera trajectory + action 的数据集仅 2-3 个，且均来自受控环境。
  3. 数据时长两极分化严重——"中等时长 + 大规模 + 多维标注" 数据集极度稀缺。
  4. 日常真实场景的高质量标注数据是社区最迫切的需求之一。
  5. 我们的数据集凭借 "日常真实场景 + GT Action + 五维 Feature 完备 + 百万级规模" 组合，占据独特且关键的位置。

推广策略建议

  • 构建标准化 Benchmark：参考 WildWorld WildBench 设计，覆盖 action/camera/long-horizon 多维评测
  • 标注格式标准化工具：开发与 SpatialVID、WildWorld、DL3DV 等的格式转换工具
  • 社区合作生态：与南大、上海 AI Lab、Meta、盛大等团队推动数据标准化
  • 分阶段开源：先开源核心子集 + 评测基准，再逐步开放完整数据
  • 开源预训练基线模型：降低社区使用门槛，加速学术影响力积累
